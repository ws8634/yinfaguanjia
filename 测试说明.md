# 银发管家 - 功能测试说明（修复版）

## 已修复的功能

### 1. 紧急呼叫功能

**已修复**：
- ✅ 120 拨打按钮现在使用与联系人相同的拨打方式 `makePhoneCall("120")`
- ✅ 紧急联系人的拨打按钮也使用 `makePhoneCall(contact.phone)`
- ✅ 所有拨打使用统一的权限检查和拨打逻辑

**拨打逻辑**：
1. 检查是否有 `CALL_PHONE` 权限
2. 如果有权限，调用 `makePhoneCallInternal` 进行拨打
3. 如果没有权限，显示权限请求对话框
4. 拨打时先播放语音提示"正在拨打 XXX"
5. 延迟 1 秒后使用 `ACTION_CALL` 直接拨打
6. 如果拨打失败（SecurityException），回退到 `ACTION_DIAL`

**测试方法**：
1. 打开"紧急呼叫"页面
2. 点击"拨打 120 急救"红色按钮
3. 或点击"120 急救"联系人卡片上的"拨打"按钮
4. 听到语音提示"正在拨打 120"
5. 1 秒后应该自动跳转到拨号界面并拨打

**如果仍然不能拨打**：
- 检查是否授予了电话权限
- 小米手机需要在"安全中心"中授予权限
- 检查拨号设置是否有呼叫限制

### 2. 语音聊天功能

**已修复**：
- ✅ 方案 2 使用 `ActivityResultLauncher` 获取语音识别结果
- ✅ 修复了之前方案 2 无法返回结果的问题
- ✅ 语音识别结果现在会显示在聊天记录中
- ✅ AI 会生成回复并语音播报

**4 种语音识别方案**：

- **方案 1**（蓝色按钮）：标准 SpeechRecognizer
  - 使用完整的 RecognitionListener 回调
  - 支持部分结果显示
  - 支持错误处理

- **方案 2**（绿色按钮）：RecognizerIntent + ActivityResultLauncher ⭐**推荐**
  - 启动系统语音识别界面
  - 使用 ActivityResultLauncher 获取结果
  - 最可靠的实现方式

- **方案 3**（橙色按钮）：延迟 SpeechRecognizer
  - 在方案 1 基础上增加 500ms 延迟
  - 可能解决某些设备的初始化问题

- **方案 7**（紫色按钮）：完整回调
  - 使用完整的 RecognitionListener
  - 设置更长的静音检测参数

**测试方法**：
1. 打开"语音聊天"页面
2. **推荐使用方案 2**（绿色按钮）
3. 点击绿色麦克风按钮
4. 授予录音权限（如果提示）
5. 系统会弹出语音识别界面
6. 说话后点击完成
7. 查看识别结果和 AI 回复

## 已知问题

1. **拨打功能**：Android 系统对自动拨打有限制，可能需要手动点击拨打按钮
2. **语音识别**：需要网络权限和录音权限，部分设备可能不支持离线识别
3. **语音播报**：需要设备支持 TTS 引擎
4. **天气查询**：当前使用模拟数据，实际应用需要接入真实天气 API

## 小米手机特殊说明

小米手机可能有以下限制：
1. **权限管理**：需要在"安全中心"中授予电话和录音权限
2. **通话限制**：可能在拨号设置中有呼叫限制
3. **后台权限**：需要在"自启动管理"中允许应用自启动
4. **悬浮窗权限**：可能需要授予悬浮窗权限
